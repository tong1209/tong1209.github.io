
## 移动端APP界面复刻设计说明

### 设计主题
专业、清晰、高效的保险金融移动应用界面，整合了客户管理、保单管理、回访照会、权益服务等核心业务流程，采用现代简约的扁平化设计风格，传达可靠与信任感。

### 适配设备
APP移动端，标准尺寸：375px × 812px
布局结构：顶部标题栏56px + 中央内容区676px + 底部导航栏80px

### 页面布局结构
应用采用底部导航（首页、工作台、客户、保单、我的）结合顶部标题栏的经典结构。中央内容区根据不同页面功能，采用列表式、卡片式或网格化布局。

        - 常用办公：4x2网格布局，包含OA办公、宏学堂等8个功能入口（图标+文字）。
        - 银保工具：单行卡片，展示“银保数字营业部”。

### 颜色方案
- **主色调/品牌色**: 绿色 (#4CAF50 或相近色值)，用于：
    - 高亮状态标签（有效、交费有效、交清）。
    - 主要操作按钮（编辑、联系、照会回复、权益按钮）。
    - 底部导航选中状态指示器。
    - 部分功能图标（OA办公、日程等）。
- **辅助色/背景色**:
    - 纯白色 (#FFFFFF): 页面及卡片背景。
    - 浅灰色 (#F5F5F5 或 #FAFAFA): 用于分割线、非选中状态背景、部分卡片底色。
    - 深灰色 (#333333 或 #666666): 主要标题、正文文字颜色。
    - 中灰色 (#999999 或 #95A5A6): 辅助文字、标签文字、占位符、图标填充色。
- **状态/警示色**:
    - 红色 (#F44336 或 #E74C3C): 用于“待复核”、“待回复”标签、超期或需紧急处理的数字（正数递交时长）。
    - 绿色 (#4CAF50): 成功状态、有效状态、负数递交时长（表示已提前）。
    - 橙色 (#FF9800): 用于部分功能图标（如美亚商旅、会议）。
    - 蓝色 (#2196F3): 用于部分功能图标（如任务、数据报表）及标签栏选中下划线。

### 排版和字体
- **字体家族**: 采用系统默认无衬线字体族，如 `-apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif`，确保清晰度和一致性。
- **字号与字重层级**:
    - **大标题/页面标题**: 较大字号，加粗 (font-weight: 600 或 bold)，如“工作台”、“个险客户”。
    - **卡片标题/模块标题**: 中等字号，常规或中等字重，如“常用办公”、“银保工具”、“主险名称”。
    - **正文/主要信息**: 标准字号，常规字重，用于保单号、客户姓名、日期等核心数据。
    - **辅助文字/标签**: 较小字号，常规字重，颜色为灰色，用于“投保人”、“生效日”、“年龄”等标签。
    - **数字/数据强调**: 关键数字（如保费、保单号）可能使用稍大或加粗字体以示强调。
    - **按钮文字**: 中等字号，加粗，颜色通常为白色（在有色按钮上）。
- **行高与间距**: 文本行高适中（如1.4-1.6），确保良好的可读性。模块间、列表项间有充足的留白（如16px, 20px, 24px）以创造呼吸感和清晰的视觉分组。

### 组件和元素
- **按钮**:
    - **主按钮 (绿色)**: 背景色为品牌绿色，文字白色，高度约44-48px，圆角（如8px或全圆角）。
    - **次要按钮 (线框)**: 白色背景，绿色边框和文字，相同高度和圆角。
    - **文本按钮/图标按钮**: 仅文字或图标，无背景，如顶部的搜索、设置图标。
- **卡片**: 白色背景，通常有细微的阴影或极细的边框 (#E8E8E8) 以提升层次感，内边距统一（如12px或16px），圆角约8px。
- **列表项**: 垂直排列，有分隔线或通过间距区分。信息布局多为左标签右内容，或左图标右文字。
- **输入框/搜索栏**: 浅灰色背景，带圆角，内部有占位符文字，右侧可能有搜索图标或按钮。
- **图标**: 简洁的线性图标或面性图标，风格统一。功能图标色彩鲜明以区分类别，系统图标多为灰色或黑色。
- **标签 (Tag)**: 小圆角矩形，用于状态显示（如红色“待回复”、绿色“有效”）。
- **进度条**: 用于显示“填写进度”，通常为细长条，已完成部分为绿色。
- **开关 (Toggle)**: 用于“加入日程”等功能，圆形滑块在轨道上滑动。
- **底部导航栏**: 高度80px，图标在上，文字在下，选中项图标和文字颜色为绿色。

### 交互效果
设计说明专注于视觉呈现与布局还原。默认不包含复杂的交互动画（如下拉刷新、页面切换动画）。静态交互状态需体现，例如：
- 按钮的`:active`状态（轻微变暗或缩小）。
- 列表项的点击态（背景色短暂变化）。
- 开关的开启/关闭状态视觉区分。
复杂的动态效果（如轮播图自动滚动、图表数据动态更新）不在本次基础复刻范围内，除非后续特别要求。

### 边界处理
由于是APP移动端界面复刻，需要为整个应用视图（375x812px）添加一个容器，并应用以下样式以模拟手机屏幕效果：
- **容器边框**: 1px 细线边框，颜色为 #CCCCCC。
- **圆角**: 容器外角应用约16px的圆角，使其更像一个手机设备。
- **投影**: 为容器添加一个适度的阴影（例如 `box-shadow: 0 4px 20px rgba(0, 0, 0, 0.1)`），以增强立体感和与网页背景的区分。
- **背景色区分**: 应用容器内部使用白色 (#FFFFFF) 作为主要背景色。HTML页面整体背景应使用浅灰色（如 #F0F0F0 或 #F5F5F5），与应用容器形成明显对比，突出显示设计内容。
